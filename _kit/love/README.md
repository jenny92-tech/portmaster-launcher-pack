# Shared LÖVE UIKit

`_kit/love` is the common controller-first UI used by every settings launcher and
by Port App Manager. Ports declare pages and fields; they do not implement another
renderer or input system.

## Files

```text
_kit/love/
  kit.lua       layout, widgets, focus, dialogs, navigation and persistence
  launcher.lua  declarative settings-launcher schema
  conf.lua      fullscreen LÖVE configuration
  ui.gptk       default keyboard mapping generated by gptokeyb

ports/<port>/love/
  main.lua                 port pages, fields and environment bindings
  launcher.sh.template     device launcher assembled for distribution
```

Ordinary game launchers use PortMaster's installed LÖVE runtime. Port App Manager
packages its own aarch64 LÖVE runtime, font and input tools so it can repair a missing
PortMaster environment.

## Public components

| API | Purpose |
| --- | --- |
| `kit.button` | Action row, including half-width and bottom-pinned actions. |
| `kit.select` / `kit.picker` | Cycling selection with stable values. |
| `kit.checkbox` | Select zero or more items. |
| `kit.switch` | Boolean setting rendered as a sliding switch. |
| `kit.info` | Read-only focusable key/value row. |
| `kit.textview` | Measured, wrapping and optionally expandable content. |
| `kit.list_item` | Compact read-only list row. |
| `kit.section` / `kit.badge` | Non-focusable grouping and status elements. |
| `kit.dialog` | Focus-trapped confirmation or information dialog. |
| `kit.toast` | Temporary operation result. |
| `kit.set_busy` | Blocking progress overlay with optional cancellation. |
| `kit.add_page` / `kit.set_page` | Create or replace page contents. |
| `kit.push_page` / `kit.back_page` | History-aware drill-down navigation. |

Common row options include stable `id`, `detail`, `disabled`, `half`, `group`, and
page-specific typography. Dynamic pages must use stable IDs so selection, scroll and
sidebar details survive sorting and refreshes.

`row_layout={mode="grid",columns=2}` creates a fixed grid. `mode="flow"` derives the
column count from `min_width`. Sections span the full width; rows sharing a visual line
use the tallest measured height.

## Application shell

`port.theme={kind="app"}` enables the Port App Manager shell: toolbar, scrolling
content, optional right action column, divider and bottom actions. A page with an empty
sidebar is a true single-pane page; no empty column or divider is reserved.

`sidebar_details` is keyed by the focused row's stable ID, never by a row index.
`sidebar_footer` displays non-focusable product/contact information. Use
`surface=false` on `kit.textview` for plain explanatory status text.

## Input contract

Widgets consume semantic actions: `up`, `down`, `left`, `right`, `confirm` and
`cancel`. `love.keypressed` translates the configured keys and calls `kit.input`.
Joystick input is disabled in `conf.lua`; gptokeyb is the single controller input path,
which prevents one physical press from being handled twice.

Ordinary launcher mappings accept either face-button convention for confirm and use
the remaining pair for return. Port App Manager carries its own mapping and explicit
on-screen buttons because physical A/B labels differ between supported firmware.

Dialogs default to their safe action. Home-page cancel handlers must open an exit
dialog and call `kit.quit()` only from the confirmed exit callback.

## State and layout

`launcher.define` derives defaults, validation, localized labels, environment output
and one-time legacy setting import from a declarative field list. Use
`launcher.resolution`, `launcher.select` and `launcher.toggle` for ordinary launchers.

State and environment files use temporary-file plus rename writes. Environment values
are shell-quoted. A failed write keeps the UI open and offers retry/stay actions.

Text is rendered on pixel-snapped coordinates. Body type uses one measured pass;
TextView height, Grid/Flow geometry and scroll bounds are cached until content,
language or expansion state changes. Call `kit.invalidate_layout()` only after editing
a row in place; `kit.set_page` invalidates automatically.

## Runtime, display and font rules

Device launch code lives in `_kit/portmaster_common.sh`, not in Lua:

- discover the newest installed `runtimes/love_*/love.txt`; do not hardcode 11.5 paths;
- use Wayland when a real socket is present and preserve that socket's runtime dir;
- otherwise allow SDL/KMSDRM and apply the PortMaster gl4es defaults;
- clear inherited settings from the opposite display path;
- resolve and validate PortMaster's shared Noto font for ordinary launchers;
- repair a recoverable shared font layout before using a per-port fallback.

Port App Manager is the exception to the runtime/font dependency: its distribution
contains the resources required to draw its repair UI before PortMaster discovery.

## Build and verify

```bash
_kit/dist_port.sh hk
_kit/dist_port.sh appmanager

bash tests/test_love_shared_components.sh
python3 tests/test_love_lua_modules.py
python3 tests/test_appmanager_environment_ui.py
```

The Python tests require `lupa`; they report `SKIP` when it is unavailable. Generated
device payloads are under `ports/<port>/dist/`. Never deploy editable templates or
`_kit` directly.
